Setting Project Options

The project options are set of advance settings that enable you to customize the code generation based on the options selected. For example, you can select a provider from dropdown from Toolbar or in Project Settings dialog box so that CodeBhagat can generate data layer class that uses the specific provider.

When you create a new projet, you should review the default settings for various code generation options for the new project.

To set project options,

1. From the Tools menu, select Options.
2. This will open the following Project Options dialog box.




General Options


By default, the system will open General tab when you open Project Options dialog box as shown above. Following are the options you can set on this tab.

On Startup Open Project Option: When you open the CodeBhagat tool, you may choose to open one of following project.
  1. Open Last Selected Project, which opens up the same project you worked last time.
  2. Show CodeBhagat Project, which opens up default project created as a sample.
  3. Show Open Project dialog box, which opens up Open Project dialog box so that you can select the specific project to work on.
Check for Updates: This will check for updates whenever you start CodeBhagat for .NET tool. If an update is available, it will prompt you to apply or not.
Code Editor You have an option to either use Code Editor provided by CodeBhagat or use the custom Code Editor. You can click on "Browse" button to select the path for your favorite code editor.
Default Language You can select the language in which you want to generate the code. The two supported languages are C# and VB.NET. The default language is C#.
You can also change this option from the toolbar.
Templates Path This is the path where you can store the specific project templates to generate your code. Each project can have their own path for templates.
Note:
  1. If you change the templates path, you will need to manually copy all the tempaltes from the default path to the new path.
  2. Although you can select different path for a specific project, you cannot change the names of the template files.
  3. You can keep separate templates for a particular project, which allows you flexibility of changing templates and keeping original templates as is.
Copy to Clipboard after Generate? When you select Yes, CodeBhagat will copy the generated source code file to Clipboard automatically. You don't have to go to the folder where the file is and manually copy the file. You can paste the generated source code file into your Visual Studio projects as soon as it generates an individual component.

Web UI Options


You may click on Web UI tab of the Project Options dialog box to set the options for the user interface layer of your project/application. Following are the options you can set on this tab.

Output Directory This is the path where the ASP.NET Pages or Web User control files will be stored for Web User Interface.
Schema Path This is the path where the schema files will be stored for Web User Interface.
Web UI Namespace The specified namespace will be used in the ASP.NET Pages or Web User control code files for Web User Interface.
Schema Namespace The specified namespace will be used in the schema files for Web User Interface.
Use Master Page If the checkbox is checked, the specified Master Page will be used during the code generation for all ASP.NET Pages.
Web Page Contains When Control is selected, the generated web page will contain the web user control reference.
When Code is selected, the generated web page will contain the code for web page.
Web Control Path The specified path will be used to reference web user controls.
Web Pages Path The specified path will be used to reference web pages.



Business Layer Options


You may click on Business Layer tab of the Project Options dialog box to set the options for the business layer of your project/application. Following are the options you can set on this tab.

Class Path This is the path where the class files will be stored for the Business Layer.
Schema Path This is the path where the schema files will be stored for the Business Layer.
Class Namespace The specified namespace will be used in the class files for the Business Layer.
Schema Namespace The specified namespace will be used in the schema files for the Business Layer.



Data Access Layer Options


You may click on Data Access Layer tab of the Project Options dialog box to set the options for the data access layer of your project/application. Following are the options you can set on this tab.

Class Path This is the path where the class files will be stored for the Data Access Layer.
Schema Path This is the path where the schema files will be stored for the Data Access Layer.
Class Namespace The specified namespace will be used in the class files for the Data Access Layer.
Schema Namespace The specified namespace will be used in the schema files for the Data Access Layer.
XSD Tool Path The specified path for xsd.exe will be used to generate the Schema files.




Database Layer Options


You may click on Database tab of the Project Options dialog box to set the options for the database layer of your project/application. Following are the options you can set on this tab.

Default Provider One of the following provider can be used from the dropdown list as a default provider.
SqlClient to use SqlClient for Database Connection using ADO.NET.
OleDb to use OleDb client for Database connection using ADO.NET.
EntLib to use Microsoft Enterprise Library for database connection.
Project Prefix The specified prefix will be used for creating a database objects. For example, nw for Northwind.
Tables Prefix The specified prefix will be used for creating a database objects. For example, tbl for Table.
Views Prefix The specified prefix will be used for a database view followed by the projet prefix. For example, nw_view_categories.
Stored Procedure Prefix The specified prefix will be used for a database stored procedure followed by the project prefix. For example, nw_sp_categories_insert
Triggers Prefix The specified prefix will be used for a database trigger followed by the project prefix. For example, nw_tri_categories_insert
SQL Output Path This is the path where the generated SQL files will be stored for the Database Layer.
Audit Trail Table Name The specified table will be used to store audit trail data through the database triggers.



Once you have defined the project settings, you can start generating the code straight from the database with the preferences you have set.
