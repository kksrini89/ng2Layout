How do I define Virtual Keys/Lookups?

To define Virtual Keys/Lookup for a particular column of a Table or View,

1. Go to Database Explorer window and Expand a particular database table or view node to see the columns list.
2. Right click on the column for which you want to generate a lookup and select Set as Virtual Key/Lookup... menu item.

This willl open the following Generate Virtual Key/Lookup window where you can define Virtual Foreign Keys or Lookup queries/controls for a column.

3. Select the Reference Table and Reference Field to define the relations for a Virtual Key and select Display Field for the Lookup from the dropdown list.
Note: Reference Field will be an internal value and Display Field will be a display value within a lookup control. 4. Enter the initial Default Value for the Lookup Control you use. This will automatically create first list item for the selected control (i.e. Dropdown List).
5. Select the Lookup Control from the dropdown list. You can choose from DropDownList, RadioButtonList, or CheckBoxList. When the tool is generating Web UI code for a specific Table or View, it will use the selected Lookup Control for the Add/Edit Web Page or Web User Control.
6. Enter Filter Expression in case you want to filter the records with specific criteria. Note: You don't need to enter the WHERE clause in the Filter Expression.
7. Click on Preview button to preview the lookup for the selected column.
8. Click on OK button to set the selected column as a Foreign Key.

The code generation tool will recognize the field as Lookup field and automatically create Lookup control while generating code for an application or individual component. If you have not defined a field as Lookup, it will appear as a TextBox entry on your web/windows forms on the generated application.
