Generate Angular 2.0 SPA Web Application

With our latest release of CodeBhagat v2.0, you can now generate Angular 2.0 SPA Web Application.

The generated application contains following features:

  • Excellent development framework for an enterprise application with best practices and application organization for Angular 2.0.
  • Responsive Web UI using Twitter Bootstrap v3.3.7
  • Contains build system using Webpack and test system using Jasmine and Karma.
  • Utilizing Protractor for end-to-end testing and Karma for unit tests.
  • Consistent Error Handling in the generated code for your components and services.
  • Mocked authentication module (will be using ASP.NET Identity in future release).
  • Consistent Error Handling in the generated code for your components and services.

To generate new Angular 2.0 SPA Web Application,

1. From the File menu, select New Application --> Angular 2.0 SPA Web Application menu item.


You will see the following dialog box appear on screen:


2. On "Generate New Angular 2.0 SPA Web Application" dialog, Enter/Select the following parameters/options you want to use to generate the application:

  • Application Name
  • Location: where you want to store the generated application
  • Language: C# or VB.NET
  • Provider: SQL or Native Client, OleDb Client or Enterprise Library.
  • Command Type: Inline SQL or Stored Procedure
  • Visual Studio Version: 2015/2013/2012/2010

3. You may select the Database (from the dropdown) and Database Objects (from the treeview) for which you want to generate the code, and click on Generate button to generate an application.
Note: This may take between 1 to 3 minutes based on number of Tables/Views you have selected.

Once the application is generated, it will be open in your choice of Visual Studio version. All you need to do is Compile and Run your application.

To Compile, select Build --> Build Solution from the menu item in Visual Studio.
To Run, click on Start button on Toolbar.

How to Compile and Run Angular 2.0 Application?

Once generated application is opened up in Visual Studio, you may follow below steps to run both Web API and Angular 2.0 application. Since Angular 2.0 application uses Web API as backend services to interact with database, you will need to run Web API application first.

To compile and run Web API application,

  1. Right click on Web API application in Solution Explorer and select “Set as Startup Project”.
  2. Click on “Start” button to compile and run your Web API application.
To compile and run Angular 2.0 application,

  1. 1. Open command prompt and go to application folder. For example, < Project Location > \ < ApplicationName > \ < ApplicationName >
  2. 2. Run "npm install" command to install node modules and dependencies, before you compile Angular 2.0 application.
  3. 3. Run "npm start" command, once node modules are installed. This will compile your application.
  4. 4. Open web browser and go to “localhost:8080” to run your application, if the browser does not open automatically.

Known Issues and Solutions

While you compile and run Angular 2.0 application, you may face certain issues based on various parameters of your environment/setup.

1) Minimum Requirement:
a. Node.JS version 6.2.2
b. NPM version 3.9.5
2) Visual Studio Version: either 2013 or 2015 is required with latest updates.


ISSUE#1: Compile errors with Angular 2.0 application:

Root Cause: Please note that upon opening Angular 2.0 application, Visual Studio may try to install node modules automatically. This may conflict with the command “npm install” on your command prompt.

Solution: To resolve this issue, you will need to delete “node_modules” folder of your Angular 2.0 application and run “npm install” from the command prompt once Visual Studio is done processing.
