Generate Stored Procedures

You can generate stored procedures by selecting a particular Table or View node from the Database Explorer. This can be done either by selecting Quick Generate or Custom Generate context menu.

Quick Generate:

You can generate the stored procedure code with just one-click action. To Quick Generate,

1. Go to Database Explorer window and expand the database nodes.
2. Right click on the database object (Table or View) to open the context menu as shown below.

3.  Select Quick Generate menu and click on Stored Procedures (CRUD) from the sub menu. This will generate the stored procedures for the following operations.

SELECT ALL To select and return all records from a table.
SELECT BY PAGE To select records by page with specified filter criteria.
SELECT BY FILTER To select records with specified filter criteria.
SELECT BY ID To select a specific record by id.
ADD To insert a record into the table.
UPDATE To update a record into the table.
DELETE To delete a record from the table.
DELETE BY FILTER To delete records from the table with specified criteria.

Following naming convetions are used while creating the stored procedures.

<ProjectPrefix>_<StoredProcedurePrefix>_<TableName>_<StoredProcedureType>

Where,
<ProjectPrefix> is the prefix you defined for project in the Tools, Options menu under Database Layer. For example, nw for Northwind project.
<StoredProcedurePrefix> is the prefix you defined for the stored procedure in the Tools, Options menu under Database Layer.
<TableName> is the table object you selected from the context menu.
<StoredProcedureType> can be Add, Edit, Delete, Select, Select by ID etc.

For example, If your table name is Customer, the following stored proedures will be created.

  • nw_sp_customer_select_all
  • nw_sp_customer_select_bypage
  • nw_sp_customer_select_filter
  • nw_sp_customer_select_byid
  • nw_sp_customer_add
  • nw_sp_customer_edit
  • nw_sp_customer_delete
  • nw_sp_customer_delete_filter

If you want to ignore the ProjecPrefix and StoredProcedurePrefix, you can choose to remove the prefixes from the Tools, Options menu.

Custom Generate:

You can choose to custom generate the stored procedure code that will allow you to select the stored procedures that you want to generate and the names you want to give to the stored procedure. To Custom Generate,

1. Go to Database Explorer window and expand the database nodes.
2. Right click on the database object (Table or View) to open the context menu as shown below.

You can use Custom Generate menu to select or enter various parameters for your specific needs before you generate the code. To Custom Generate,

1. Go to Database Explorer window and expand the database nodes.
2. Right click on the database object (Table or View) to open the context menu as shown below and select Generate menu.

3. Click on the Stored Procedures (CRUD) or Stored Procedures (INSERT/UPDATE) to generate specific stored procedures from the sub menu.

When you select Stored Procedures (CRUD), this will open the following popup window where you can select specific stored procedures and define the names of the stored procedures before you generate the code.

When you select Stored Procedures (INSERT/UPDATE), this will open the following popup window where you can select specific parameters to generate INSERT/UPDATE stored procedures code.
