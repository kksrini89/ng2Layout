CodeBhagat for .NET vs. Other Code Generators

Why CodeBhagat for .NET?

Here is comparision between CodeBhagat for .NET and other code generators:

CodeBhagat for .NET Other Code Generators
Choice of Technology Allows you to generate code for any of the following technologies:
  • Angular 2.0/TypeScript
  • AngularJS 1.x
  • ASP.NET MVC
  • ASP.NET Web Forms
  • Web API Services
  • WCF/WCF RESTful Services
  • WPF using MVVM Pattern
 Allows you to generate code only for 1 or 2 technologies: ASP.NET or ASP.NET MVC
Choice of Language Allows you to generate code either in C# or VB.NET Allows you to generate code only in 1 language.
Choice of Database Allows you to generate code straight from one of the 8 supported databases:
  • SQL Server
  • SQL Azure
  • Oracle
  • MySQL
  • SQL Standalone Databases (*.mdf)
  • SQL Mobile/CE (*.sdf)
  • SQL Express
  • MS-Access
 Most of the code generators does not support Oracle, SQL Mobile/CE, MySQL and MS-Access.
Entire Application vs Individual Component Allows you to generate code for an entire application or individual components during development cycle. So, when the schema of a particular table/view changes or new table is introduced in the database, I can generate an individual component (UI, Business, DAL, Service Layer or Database Layer) and apply the same in my existing application/project. Most of the code generators does not let you generate individual components while you are in the middle of development cycle and code generation software becomes useless during this cycle.
Choice of Database Provider Allows you to choose the Database Provider: Native Provider (SqlClient for SQL databases, MySqlClient for MySQL, OracleClient for Oracle etc.), OleDb Client as well as Enterprise Library. Many enterprise applications are utilizing Enterprise Library due to dynamic nature of their backend databases. Most of the code generators do not allow the user to pick the Database Provider. This becomes a limitation for many enterprise level applications.
Customization of Code Templates Easy to change the code generation templates and re-generate an application or individual component. You can make changes to the default code blocks such as Error Handler, Method Signatures etc. Difficult to change the code and code generation templates once the application is already generated.
